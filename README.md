# Чек-лист по использованию ИИ — Telegram

Два готовых способа. **Вариант 1 не требует никакого хостинга** — нужен только токен бота.

## ⚠️ Сначала — безопасность

Токен, отправленный ранее в переписке, считается **скомпрометированным**.
Откройте **@BotFather → /revoke** (или `/token`) и выпустите новый.
Токен нигде не вставляйте в код — только в переменную окружения.

---

## Вариант 1 — Автономный бот (без хостинга) ⭐ рекомендуется

Бот сам присылает анимацию и показывает чек-лист прямо в чате: пункты
отмечаются нажатием кнопок (✅ / ⬜), прогресс сохраняется для каждого пользователя.

**Файлы:** `checklist_bot.py`, `data-flow.mp4`, `requirements.txt`

```bash
pip install -r requirements.txt

export BOT_TOKEN="НОВЫЙ_ТОКЕН_ОТ_BOTFATHER"     # Windows: set BOT_TOKEN=...
python checklist_bot.py
```

Откройте бота в Telegram → `/start`. Готово — никакого сервера и HTTPS не нужно.

> Файл `data-flow.mp4` должен лежать рядом с `checklist_bot.py` (он уже здесь).
> Отметки хранятся в `state.json`, который бот создаёт автоматически.

---

## Вариант 2 — Красивая версия как Mini App (нужен HTTPS-хостинг)

Открывает полноценный визуальный чек-лист (с фоновым видео и анимацией) внутри Telegram.

**Файлы:** `checklist-standalone.html` (всё в одном файле), `bot.py`

1. Разместите `checklist-standalone.html` по HTTPS — например перетащите его на
   **Netlify Drop** (https://app.netlify.com/drop) или залейте в **GitHub Pages**.
2. Запустите:
   ```bash
   export BOT_TOKEN="НОВЫЙ_ТОКЕН"
   export WEBAPP_URL="https://ваш-домен/checklist-standalone.html"
   python bot.py
   ```

Можно совместить: если задать `WEBAPP_URL` и для **Варианта 1**, в автономном
боте появится дополнительная кнопка «🌐 Открыть красивую версию».

## Что внутри чек-листа

- **Правильное использование ИИ** — 8 привычек осознанного пользователя
- **Анатомия сильного промпта** — 7 элементов хорошего запроса
- **Чего избегать** — 6 частых ошибок
